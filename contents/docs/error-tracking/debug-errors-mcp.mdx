The [PostHog MCP server](/docs/model-context-protocol) exposes function calling tools to any MCP client, enabling AI agents to interact with PostHog's API via the MCP protocol.

When combining our MCP server with Error Tracking, your AI agents can take actions based on PostHog data which unlocks powerful, autonomous debugging capabilities.

For example, with MCP, your agents can:

- **Triage issues** – Fetch the latest issues with details like status, frequency, users affected, last seen, etc.
- **Inspect error details** – Access full stack traces, error messages, and relevant metadata.
- **Explain or reproduce errors** – Identify failure points and perform root cause analysis using the captured stack trace context.
- **Debug and create fixes** – Propose or even generate code fixes based on the error details.
- **Update issue status** – Mark issues as resolved, archived, suppressed, or pending release directly from your code editor.
- **Create assignment rules** – Set up automatic issue assignment rules with filters and an assignee directly from your agent.
- **Update grouping rules** – Modify filters on existing grouping rules to control which exceptions get grouped together.
- **Get setup recommendations** – Surface suggestions to improve your error tracking setup, including alerts, rate limits, source maps, and managing long-running issues.

All of this happens directly inside the MCP client, like Cursor, Windsurf, or Claude Code, so you can investigate and resolve product issues without ever leaving the code editor.

Try these example questions and directives with your MCP-enabled agent:

- `Show me my most common errors.`
- `What's the full stack trace for the most recent error?`
- `Create a fix and show me how to reproduce the error with the highest severity.`
- `Which error is causing the most crashes in production?`
- `Mark this issue as resolved.`
- `Set all high-severity errors from today to suppressed.`
- `Create an assignment rule that assigns all TypeError exceptions to the backend team.`

- `What recommendations do you have for improving my error tracking setup?`
- `Help me set up alerts for new error issues.`
- `Do I need to configure rate limits for error tracking?`

## Install the MCP server

To start debugging with PostHog MCP, install the MCP server in your preferred MCP client.

The recommended way is to use the [AI wizard](/docs/ai-engineering/ai-wizard), which supports setup for agents like Claude, Cursor, Windsurf, VS Code, and more. The wizard guides you through the installation step by step.

```bash
npx @posthog/wizard mcp add
```

You can also install the MCP server by [configuring it manually](/docs/model-context-protocol#manual-install).

Check out the [MCP server documentation](/docs/model-context-protocol) for more detailed instructions and information.

## Get setup recommendations

The recommendations tool surfaces suggestions to help you optimize your error tracking configuration. When you ask the agent for recommendations, it analyzes your current setup and provides actionable guidance for each issue it finds.

The agent can help you with:

- **Alerts** – Set up notifications for new issues, reopened issues, or volume spikes so you're alerted to problems as they happen.
- **Rate limits** – Configure project-wide and per-issue ingestion limits to control costs and prevent noisy issues from overwhelming your error budget.
- **Source maps** – Detect when stack traces aren't resolving properly and guide you through uploading source maps with the setup wizard.
- **Long-running issues** – Identify stale issues that have been active for over a week and help you triage or suppress them.

The agent works with other MCP tools to implement recommendations directly. For example, it can create alerts, update rate limit settings, or help you resolve long-running issues – all without leaving your code editor.
