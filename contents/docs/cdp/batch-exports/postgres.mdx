Batch exports can be used to export data to a Postgres table.

## Requirements

To create a batch export to Postgres, you need:

- A Postgres database accessible from the internet (internal/private IPs are not allowed)
- SSL/TLS enabled on your Postgres instance. This [provides protection against various types of attacks](https://www.postgresql.org/docs/current/libpq-ssl.html#LIBPQ-SSL-PROTECTION)
- A Postgres user with the appropriate permissions (see below)

## Setup instructions

### 1. Create a Postgres user

Create a user with table creation privileges. If the destination table doesn't exist when a batch export runs, PostHog creates it. This requires `CREATE TABLE` and `USAGE` permissions. The destination table also requires `INSERT`, `SELECT`, and `UPDATE` permissions.

Block PostHog from accessing other tables by creating a dedicated schema and granting access only to that schema:

```sql runInPostHog=false
CREATE USER posthog WITH PASSWORD 'insert-a-strong-password-here';
CREATE SCHEMA posthog_exports;
GRANT CREATE ON SCHEMA posthog_exports TO posthog;
GRANT USAGE ON SCHEMA posthog_exports TO posthog;
```

### 2. Gather connection details

You need the following information:

- **Host** - The hostname or IP address of your Postgres server
- **Port** - The TCP port (default is 5432)
- **User** - The username created in step 1
- **Password** - The password for the user

## Creating the batch export

1. In PostHog, go to [Data management > Destinations](https://app.posthog.com/data-management/destinations).
2. Click **+ New destination**.
3. Select **Postgres** as the batch export destination.
4. Create a Postgres integration (if you haven't already):
   - Click **Configure new PostgreSQL connection**
   - Enter your **Host** (from [step 2](#2-gather-connection-details))
   - Enter your **Port** (default 5432)
   - Enter your **User** (from [step 1](#1-create-a-postgres-user))
   - Enter your **Password** (from [step 1](#1-create-a-postgres-user))
   - Configure SSL verification (see [SSL configuration](#ssl-configuration))
   - Click **Connect**
5. Fill in the destination settings:
   - **Database** - The name of the Postgres database
   - **Schema** - The schema name (e.g., `posthog_exports`)
   - **Table name** - The name of the table where data will be exported
6. Configure any additional export settings (interval, model, etc.).
7. Click **Create**.

The batch export schedules its first run at the start of the next period.

### SSL configuration

When creating a Postgres integration, you can configure how PostHog verifies the server's SSL certificate:

- **No** - Skip certificate verification. Use this for development or when your server uses a self-signed certificate you don't want to verify.
- **Verify certificate authority** - Verify the server's certificate is signed by a trusted CA. You can either:
  - Check **Use the system certificate authorities** to use standard trusted CAs
  - Upload a **Root certificate** file (.crt, .pem, .cer, or .ca-bundle) for custom or self-signed CAs
- **Verify certificate authority and server hostname** - Same as above, but also verifies the server's hostname matches the certificate. This is the most secure option.

## Models

> **Note:** New fields may be added to these models over time. To maintain consistency, these fields are not automatically added to the destination tables. If a particular field is missing in your Postgres tables, you can manually add the field, and it will be populated in future exports.

### Events model

This is the default model for Postgres batch exports. The schema of the model as created in Postgres is:

| Field       | Type                       | Description                                                               |
| ----------- | -------------------------- | ------------------------------------------------------------------------- |
| uuid        | `VARCHAR(200)`             | The unique ID of the event within PostHog                                 |
| event       | `VARCHAR(200)`             | The name of the event that was sent                                       |
| properties  | `JSONB`                    | A JSON object with all the properties sent along with an event            |
| elements    | `JSONB`                    | This field is present for backwards compatibility but has been deprecated |
| set         | `JSONB`                    | A JSON object with any person properties sent with the `$set` field       |
| set_once    | `JSONB`                    | A JSON object with any person properties sent with the `$set_once` field  |
| distinct_id | `VARCHAR(200)`             | The `distinct_id` of the user who sent the event                          |
| team_id     | `INTEGER`                  | The `team_id` for the event                                               |
| ip          | `VARCHAR(200)`             | The IP address that was sent with the event                               |
| site_url    | `VARCHAR(200)`             | This field is present for backwards compatibility but has been deprecated |
| timestamp   | `TIMESTAMP WITH TIME ZONE` | The timestamp associated with an event                                    |

### Persons model

The schema of the model as created in Postgres is:

| Field                      | Type                       | Description                                                                                                                                        |
| -------------------------- | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| team_id                    | `INTEGER`                  | The id of the project (team) the person belongs to                                                                                                 |
| distinct_id                | `TEXT`                     | A `distinct_id` associated with the person                                                                                                         |
| person_id                  | `TEXT`                     | The id of the person associated to this (`team_id`, `distinct_id`) pair                                                                            |
| properties                 | `JSONB`                    | A JSON object with all the latest properties of the person                                                                                         |
| person_distinct_id_version | `INTEGER`                  | Internal version of the person to `distinct_id` mapping associated with a (`team_id`, `distinct_id`) pair, used by batch export in merge operation |
| person_version             | `INTEGER`                  | Internal version of the person properties associated with a (`team_id`, `distinct_id`) pair, used by batch export in merge operation               |
| created_at                 | `TIMESTAMP WITH TIME ZONE` | The timestamp when the person was created                                                                                                          |

The Postgres table will contain one row per `(team_id, distinct_id)` pair, and each pair is mapped to their corresponding `person_id` and latest `properties`.
